# 90-дневная гарантия документов с автозаполнением пробелов

> Generated by Deep Copilot · Plan mode
> 2026-07-04T10:47:59.133Z

## 目标 / Goal
Гарантировать минимум 90 дней документов в index_documents. При первой синхронизации — полный бэкфилл. При каждой последующей — проверка пробелов за 90 дней и докачка недостающих дней, плюс обычная синхронизация за 24 часа.

## 方案 / Approach
Добавить `ensure90DaysOfDocuments` в cron.ts: запрашивает DISTINCT date(close_date) за 90 дней, сравнивает с полным списком дней, для каждого диапазона пропусков вызывает getDocumentsIndexForShops. Заменить старый syncDocuments на: ensure90Days + 24h-синхронизация. Убрать проверку "таблица пуста".

## 步骤 / Steps
1. **Добавить ensure90DaysOfDocuments в cron.ts** — Добавить ensure90DaysOfDocuments: запрос DISTINCT date(close_date) за 90 дней, поиск пропусков, группировка в диапазоны, вызов getDocumentsIndexForShops для каждого диапазона
2. **Заменить syncDocuments** — Убрать старую логику «таблица пуста → 30 дней», заменить на ensure90Days + 24h
3. **Проверить, что пробелы заполняются** — Через curl или через лог wrangler

## 涉及文件 / Affected files
- `packages/backend/src/sync/cron.ts` — Добавить ensure90DaysOfDocuments, заменить syncDocuments

## 风险与缓解 / Risks
- При большом количестве пропущенных дней (холодный старт) может быть много запросов к Evotor API — нужна задержка между диапазонами
- Несколько магазинов × 90 дней = потенциально большой объём данных
- Пробелы проверяются каждый cron-вызов (каждые 3 мин) — при отсутствии пробелов это лёгкий запрос DISTINCT date()
