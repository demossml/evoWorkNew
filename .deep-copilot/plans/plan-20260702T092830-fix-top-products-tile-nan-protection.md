# Fix Top Products tile + NaN protection

> Generated by Deep Copilot · Plan mode
> 2026-07-02T06:28:30.966Z

## 目标 / Goal
Плитка «Топ продуктов» на главной странице не загружается — показывает пустое состояние. Причина: виджет использует `fetchFinancialMetrics` → `sales-garden-report`, который не возвращает поле `topProducts`. Также есть проблема с NaN в форматировании валюты в RevenueCard.

## 方案 / Approach
1. Добавить в `evotor/utils.ts` функцию `getTopProductsFromD1`, которая агрегирует топ-10 товаров из `index_documents` за период и возвращает массив `ProductData[]`.
2. Модифицировать эндпоинт `sales-garden-report`, чтобы он дополнительно вычислял `topProducts` через D1.
3. Защитить все `formatMoney`/`formatCurrency` от NaN — `Intl.NumberFormat.format(NaN)` выдаёт строку `"NaN"`, что отображается как `"NaN ₽"`.

## 步骤 / Steps
1. **Добавить getTopProductsFromD1 в evotor/utils.ts** — Агрегирует SELL/PAYBACK транзакции из index_documents, группирует по productName, сортирует по netRevenue DESC, возвращает top-10.
2. **Добавить topProducts в ответ sales-garden-report** — В api.ts вызвать getTopProductsFromD1 и включить результат в JSON-ответ.
3. **Защитить все formatMoney от NaN** — Добавить проверку !Number.isFinite(value) во все функции: formatCurrency.tsx, planService.ts, dashboardSummaryModel.ts, DashboardSummaryAiSection.tsx, StockHealthWidget.tsx, SalestReportForThePeriod.tsx.
4. **Проверить компиляцию и тесты** — npx tsc --noEmit для frontend и backend, npx vitest run.

## 风险与缓解 / Risks
- getTopProductsFromD1 сканирует всю index_documents за период — при большом объёме данных может быть медленным. Решение: лимит 10 записей + пагинация не требуется.
- sales-garden-report уже выполняет запросы к D1 — добавление ещё одного запроса увеличит latency на ~100-200ms.
