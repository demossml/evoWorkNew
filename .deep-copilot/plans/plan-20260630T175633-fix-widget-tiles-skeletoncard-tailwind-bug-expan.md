# Fix widget tiles — SkeletonCard Tailwind bug + expand animations + tile consistency

> Generated by Deep Copilot · Plan mode
> 2026-06-30T14:56:33.184Z

## 目标 / Goal
Fix the widget tiles on the Home page: (1) the SkeletonCard loading state has a broken Tailwind dynamic class that prevents colored border accents from rendering, (2) the expand/collapse detail panel has no animation — it just pops in, (3) multiple widgets duplicate identical expand/collapse boilerplate without a shared wrapper, (4) loading/empty states are inconsistent across widgets.

## 方案 / Approach
Fix the critical SkeletonCard Tailwind bug first, then wrap the shared expand/collapse pattern into one reusable TileWrapper component so all 6 widgets (Revenue, SalesTempo, Finance, BestShop, TopProduct, Accessories) use identical styling and animation. Add a smooth height transition for the detail panel. Finally, unify loading/empty states.

## 步骤 / Steps
1. **Fix SkeletonCard broken border accent (Tailwind dynamic class anti-pattern)** — Replace dynamic `border-l-${tone}-500` with a static map of complete Tailwind classes.
2. **Extract shared expand/collapse wrapper into TileWrapper component** — Create a TileWrapper component (wrapper + card div + detail panel slot) with framer-motion AnimatePresence. Replace ~80 lines of duplicated JSX per widget.
3. **Add smooth height animation for detail panel expand/collapse** — Use motion.div with animate={{ height: 'auto' }} / exit={{ height: 0 }} for the detail panel.
4. **Unify loading/error/empty states across all 6 widgets** — RevenueWidget: loading→SkeletonCard, error→ErrorState, empty→EmptyTile, normal→RevenueCard. Apply same pattern to all widgets.
5. **TypeCheck + vitest + visual smoke check** — Run tsc --noEmit and vitest to confirm nothing breaks. Check the page in browser.
