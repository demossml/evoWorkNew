# Seller DNA: same-weekday per-hour comparison table + time granularity control

> Generated by Deep Copilot · Plan mode
> 2026-07-08T08:56:49.935Z

## 目标 / Goal
Переработать сравнение продавцов в SellerDNAWidget: вместо агрегированных метрик показывать таблицу с разбивкой по часам за последние 5 одинаковых дней недели. Добавить переключение гранулярности времени (час → 30 мин → 15 мин → 5 мин → 1 мин), чтобы на телефоне можно было увидеть, как долго продавец отсутствует на рабочем месте (мёртвое время).

## 方案 / Approach
Два этапа:

**Этап 1 — Новая таблица сравнения по часам (same-weekday).**
Текущий weekday-compare возвращает `WeekdayCompareProfile` с `hourlyRevenue` (усреднённым по дням). Нужно:
- В бэкенде: новый эндпоинт `/api/sellers/hourly-compare` — принимает `targetDate`, `weeksBack` (по умолчанию 5), `sellerIds`, `granularityMinutes` (60/30/15/5/1). Для каждого выбранного продавца запрашивает SELL-документы за совпадающие дни недели, группирует по временным слотам, считает среднюю выручку и количество чеков на слот, флаг активности (были ли продажи).
- На фронтенде: новая вкладка или режим в `SellerComparisonView` — таблица, где строки = временные слоты (например, 10:00–10:59), колонки = выбранные продавцы. Ячейка показывает: сумму выручки (или "—") + цветовой индикатор (зелёный = активен, красный = мёртвое время).

**Этап 2 — Переключатель гранулярности.**
- Добавить в интерфейс ряд кнопок: 1м | 5м | 15м | 30м | 1ч.
- При переключении перезапрашивать данные с новым `granularityMinutes`.
- На мелких гранулярностях (1м, 5м) таблица становится вертикально скроллируемой. Для мобильного — показывать не более 3–4 продавцов одновременно.
- Под таблицей — сводка: общее мёртвое время каждого продавца за день, максимальная длительность отсутствия.

**Технические детали:**
- Бэкенд: новый файл `packages/backend/src/services/sellerHourlyCompare.ts`
- Эндпоинт: `GET /api/sellers/hourly-compare?targetDate=...&weeksBack=5&sellerIds=...&granularityMinutes=60`
- Кэширование в памяти (5 минут)
- Фронтенд: новый хук `useHourlyCompare`, новый компонент `HourlyCompareTable` внутри `SellerComparisonView`
- Интеграция в существующий UI виджета

## 步骤 / Steps
1. **Backend: создать sellerHourlyCompare.ts — запрос SELL-документов и группировка по временным слотам** — Принимает targetDate, weeksBack, sellerIds, granularityMinutes. Для каждого продавца: собирает SELL-доки за все совпадающие дни недели, группирует close_date по временному слоту (округление до granularityMinutes), считает avgRevenue, avgChecks, active (bool) на слот.
2. **Backend: добавить эндпоинт GET /api/sellers/hourly-compare в api.ts** — Валидация параметров, кэш на 5 минут, обработка ошибок.
3. **Frontend: новый хук useHourlyCompare.ts** — useQuery с ключом [sellers, hourly-compare, targetDate, weeksBack, sellerIds, granularity]. Возвращает { slots: HourlySlot[], sellers: HourlySellerProfile[] }.
4. **Frontend: компонент HourlyCompareTable — таблица время × продавцы** — Строки = временные слоты, колонки = продавцы. Ячейка: выручка + цвет (зелёный/красный). Мобильная адаптация: горизонтальный скролл, максимум 3–4 продавца.
5. **Frontend: переключатель гранулярности + сводка мёртвого времени** — Кнопки 1м/5м/15м/30м/1ч. Под таблицей: общее мёртвое время, макс. длительность отсутствия на продавца.
6. **Frontend: интеграция в SellerComparisonView / SellerDNAWidget** — Новая вкладка или замена текущего weekday-режима. Сохранение обратной совместимости с существующим режимом.
7. **Проверка: tsc в обоих пакетах, ручное тестирование** — Убедиться, что старый функционал сравнения не сломан.

## 风险与缓解 / Risks
- Большой объём данных на 1-минутной гранулярности: 12 часов × 60 = 720 строк. Решение: виртуальный скролл или пагинация.
- Медленный запрос к D1 при большом количестве SELL-документов. Решение: кэширование на 5 минут, передача списка конкретных дат.
- Мобильный экран: 3–4 продавца × N слотов. Решение: горизонтальный скролл, фиксированный первый столбец (время).
- Обратная совместимость: старый режим сравнения должен остаться доступным. Решение: добавить как новую вкладку, не заменять.
