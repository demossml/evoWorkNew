# Self-host backend on Node.js — local SQLite + FS adapters + server.ts

> Generated by Deep Copilot · Plan mode
> 2026-07-04T16:20:51.577Z

## 目标 / Goal
Развернуть бэкенд на Linux без Cloudflare Workers. Заменить D1 на локальный SQLite, R2 на локальную файловую систему. Запускать через Node.js одной командой.

## 方案 / Approach
Создать три файла:
1. `src/adapters/local-db.ts` — класс LocalD1Database, эмулирует D1 API (prepare/bind/all/first/run/batch) на better-sqlite3
2. `src/adapters/local-storage.ts` — класс LocalR2Bucket, эмулирует R2 API (put/get) на локальной ФС
3. `server.ts` — точка входа: Hono на @hono/node-server, внедряет локальные адаптеры вместо Cloudflare-биндингов

Минимально инвазивно: адаптеры реализуют ровно те методы D1/R2, которые используются в коде. Код api.ts, utils.ts и т.д. не меняется.

## 步骤 / Steps
1. **Создать src/adapters/local-db.ts** — Класс LocalD1Database + LocalD1PreparedStatement — .prepare().bind().all(), .first(), .run(), .raw(), .batch()
2. **Создать src/adapters/local-storage.ts** — Класс LocalR2Bucket — .put(key, arrayBuffer, metadata), .get(key) → {body, httpMetadata}
3. **Создать server.ts — точка входа** — Импорт Hono, @hono/node-server, внедрение адаптеров в контекст (c.env, c.var), serve({ fetch, port })
4. **Установка зависимостей + проверка запуска** — npm i better-sqlite3 @hono/node-server, npx tsc --noEmit, node --experimental-strip-types server.ts

## 风险与缓解 / Risks
- D1Result типы используются в коде — адаптер должен возвращать идентичную структуру { results: [...] } / { success: true }
- c.env (Bindings) и c.var (Variables) типы завязаны на Cloudflare — нужен новый тип или any
- Миграции Drizzle генерируют SQL для D1 — нужно применить их вручную через db.exec()
