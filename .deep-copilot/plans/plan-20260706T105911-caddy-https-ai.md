## 目标 / Goal
Создать полную деплой-инструкцию и скрипты для развёртывания проекта на Linux-сервере с Caddy (автоматический HTTPS), своим доменом, одной командой. Инструкция должна быть понятна другому AI-агенту.

## 方案 / Approach
Переписать деплой-скрипты с nginx на Caddy, упростить до одного systemd-сервиса (cron уже встроен в server.ts), и создать корневой DEPLOY.md с пошаговой инструкцией на русском языке, включая заполнение переменных окружения (токены, домен) и проверку результата.

## 步骤 / Steps
1. **Переписать deploy/evo-app.service** — Убрать хардкод-токены, заменить на плейсхолдеры __EVOTOR_TOKEN__ и т.д. Убрать evo-cron.service (не нужен — cron внутри server.ts).
2. **Создать deploy/Caddyfile** — Конфиг Caddy: проксирование /api/* и /health на localhost:3000, раздача статики из ../frontend/dist, авто-HTTPS через домен.
3. **Переписать deploy/setup.sh** — Полный автосетап: Node.js, pnpm, Caddy, сборка фронтенда, env-переменные через .env, systemd-сервис.
4. **Создать DEPLOY.md в корне** — Пошаговая инструкция на русском для AI-агента: 1) клонировать репо, 2) заполнить .env, 3) sudo bash deploy/setup.sh, 4) проверить. Явно указать что заменять: домен, токены.
5. **Удалить evo-cron.service и cron-runner.ts** — Они больше не нужны — все cron-задачи внутри server.ts.
6. **Обновить .gitignore** — Добавить data/ для production-окружения.
7. **Запушить в GitHub** — Коммит с описанием деплой-изменений.

## 风险与缓解 / Risks
- Caddy должен быть установлен на сервере (apt install caddy) — скрипт это делает
- Домен должен быть направлен A-записью на IP сервера до запуска — иначе Caddy не выпустит сертификат
- Node.js v22+ нужен для --experimental-strip-types, скрипт устанавливает через nodesource
- Токены Evotor и Telegram должны быть реальными — AI-агент должен их вписать в .env
- .env с токенами не должен коммититься — .gitignore уже содержит .env
