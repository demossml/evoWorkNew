# Fix: Accessories tile, Sales Tempo, Best Shop — all show zeros

> Generated by Deep Copilot · Plan mode
> 2026-07-04T09:20:05.094Z

## 目标 / Goal
Исправить три виджета на главной странице, которые показывают нули или пустые данные: плитка Аксессуары, Темп продаж, Лучший магазин.

## 方案 / Approach
Корень всех трёх проблем — отсутствующие или замоканные пустышками эндпоинты в бэкенде. Фронтенд-код и хуки написаны правильно, но сервер возвращает пустые ответы (моки) или 404.

## 步骤 / Steps
1. **Создать эндпоинт accessoriesSales/:role/:userId** — Вызывает getAccessoriesSalesFromD1(db, since, until, shopNameMap). Функция уже написана в evotor/utils.ts — нужно только добавить роут в api.ts.
2. **Создать эндпоинт hourly-plan-fact** — GET /api/analytics/revenue/hourly-plan-fact?date=X&shopName=Y. Нужно реализовать логику загрузки плана продаж из D1 (таблица plan уже есть) и факта из index_documents, посчитать план-факт по часам.
3. **Починить dashboard-home-insights (BestShopWidget)** — Сейчас возвращает мок с dayLeader={name:'',revenue:0}. Нужно реализовать реальную логику: посчитать лучший магазин за день и за неделю на основе данных из D1.
4. **Добавить синхронизацию плана продаж** — Таблица plan уже создаётся (createPlanTable), но нет scheduled sync для наполнения её данными. Добавить задачу 'plan' в sync-роутер и local-scheduler.

## 涉及文件 / Affected files
- `packages/backend/src/api.ts` — Добавить 3 недостающих эндпоинта: accessoriesSales, hourly-plan-fact, исправить dashboard-home-insights
- `packages/backend/src/evotor/utils.ts` — Уже содержит getAccessoriesSalesFromD1 — используется новым эндпоинтом
- `packages/backend/src/sync/cron.ts` — Добавить sync задачу для плана продаж
- `packages/backend/scripts/local-scheduler.ts` — Добавить интервал синхронизации плана

## 风险与缓解 / Risks
- Аксессуары идентифицируются по названию группы (LIKE '%АКСЕСУАР%') — если группы названы иначе, данных не будет
- hourly-plan-fact требует плана продаж в таблице plan — если план не заполнен, всё равно покажет нули
- BestShop требует данных о продажах за 7 дней — если index_documents не синхронизирован за всю неделю, лидер будет неполным
