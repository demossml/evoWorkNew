# Тест синхронизации за один день

## 目标 / Goal
Протестировать синхронизацию документов за сегодняшний день (один день), чтобы увидеть: доходят ли запросы до Evotor API, какой приходит ответ, сколько документов, какие ошибки.

## 方案 / Approach
Создать лёгкий standalone-скрипт `test-sync-today.ts`, который использует те же Evotor-токен и класс Evotor, что и сервер, но запрашивает строго один день. Скрипт логирует каждый шаг: магазины, HTTP-статусы, количество документов, ошибки. Запустить через `npx tsx`, проанализировать вывод.

## 步骤 / Steps
1. **Создать test-sync-today.ts** — Скрипт в packages/backend, который: (1) создаёт new Evotor(token), (2) получает список магазинов, (3) форматирует today 00:00 → today 23:59 вручную (в обход formatDateWithTime, чтобы исключить баг формата), (4) вызывает getDocumentsBySellPayback для каждого магазина, (5) выводит: статус, количество документов, типы, ошибки
2. **Запустить скрипт через npx tsx** — npx tsx test-sync-today.ts из packages/backend. Смотреть stdout на предмет HTTP-ошибок, 429, формата даты
3. **Проанализировать результаты** — Смотрим: сколько магазинов ответили, сколько документов каждого типа, какие ошибки (400 — формат даты, 429 — лимит, 401/403 — токен)

## 涉及文件 / Affected files
- `packages/backend/test-sync-today.ts` — Новый standalone тестовый скрипт (создать)

## 风险与缓解 / Risks
- Evotor API может вернуть 400 если формат даты всё ещё не подходит — тогда нужно скорректировать формат в скрипте
- Может быть 429 rate-limit, если недавно были запросы от сервера — скрипт должен показать это явно
- Если за сегодня нет реальных продаж — documents будет 0, это нормально, но неинформативно; можно попробовать вчера
