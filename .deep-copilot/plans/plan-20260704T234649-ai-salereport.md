## 目标 / Goal
Починить ошибку «Не удалось получить AI анализ» на странице отчёта по продажам (SaleReport).

## 方案 / Approach
Фронтенд вызывает `POST /api/ai/insights` с телом `{ startDate, endDate, shopUuid }`, но на бэкенде такого роута нет. Нужно добавить роут, который получает документы продаж за указанный период/магазин, прогоняет их через DeepSeek API (используя существующую инфраструктуру `createTask` из `ai/index.ts`) и возвращает структуру `{ insights, anomalies, patterns }`. Параллельно нужно убедиться, что сервер запущен с валидным `DEEPSEEK_API_KEY`.

## 步骤 / Steps
1. **Запустить сервер бэкенда** — Сейчас не запущен, нет автосинхронизации. Нужен для тестирования.
2. **Добавить POST /api/ai/insights в api.ts** — Вызывает analyzeDocsAnomaliesTask + analyzeDocsInsightsTask + analyzeDocsPatternsTask, собирает результаты в AiInsightsData.
3. **Проверить работу эндпоинта** — После исправления вызвать эндпоинт curl'ом
4. **Проверить ключ DeepSeek API** — Убедиться, что DEEPSEEK_API_KEY не 'dummy' и API отвечает
