## 目标 / Goal
Плитка «Аксессуары» должна работать от D1, показывать сумму аксессуаров за период, при разворачивании — список всех аксессуаров с разбивкой по магазинам. Тоггл «Акс./Не акс.» должен реально переключать данные.

## 方案 / Approach
1. Переписать бэкенд-эндпоинт `accessoriesSales` с Evotor API на D1. Новая функция `getAccessoriesSalesFromD1` будет запрашивать `index_documents`, классифицировать товары как аксессуары / не аксессуары через `shopProduct` (группа с названием, содержащим "аксессуар"), и возвращать обе категории.
2. Исправить фронтенд: summary stats должны учитывать текущий scope (акс/не акс), элементы списка должны содержать `shopName` для фильтрации.

## 步骤 / Steps
1. **Добавить getAccessoriesSalesFromD1 в evotor/utils.ts** — Запрашивает index_documents за период, для каждого товара определяет группу через shopProduct, разделяет на аксессуары и не аксессуары. Возвращает byShop/total для обеих категорий.
2. **Переписать эндпоинт accessoriesSales на D1** — Убрать вызовы evo.getDocuments/evo.getShopName, использовать getAccessoriesSalesFromD1. Обрабатывать и SUPERADMIN, и обычного пользователя.
3. **Исправить buildAccessoriesSummaryStats для учёта scope** — Передавать список товаров (не весь data.total), чтобы статистика пересчитывалась при переключении акс/не акс.
4. **Проверить фронтенд: shopName в total элементах** — Убедиться что items содержат shopName для корректной фильтрации по магазину.
5. **Проверить компиляцию и тесты** — npx tsc --noEmit, npx vitest run
