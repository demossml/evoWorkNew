## 目标 / Goal
Три улучшения Seller DNA: (1) почасовое сравнение со средним по тому же дню недели за 4 прошлые недели, (2) сравнение продавцов по одинаковым дням недели с ручным выбором периода, (3) фильтр рейтинга: today-only / все продавцы за период.

## 方案 / Approach
## 1. Почасовой бенчмарк (4-week same-weekday average)

Сейчас `storeAvgHourlyRevenue` = сумма всей выручки магазина / календарные дни. Это некорректно: сравниваем понедельник со средним по всем дням недели.

**Решение:** новый параметр `benchmarkWeekday` в `computeSellerAdvancedStats`. Если передан, бэкенд для каждого часа считает среднюю выручку магазина ТОЛЬКО за тот же день недели за 4 предыдущие недели.

**Бэкенд** (`sellerAdvancedStats.ts`):
- Добавить `benchmarkWeekday?: number` (0=Пн..6=Вс) в params
- Новый приватный метод `computeWeekdayBenchmark(db, since, until, shopId, weekday)`:
  - Для каждой из 4 предыдущих недель найти дату того же дня недели
  - Запросить SELL-документы за эти 4 даты
  - Сгруппировать по часам, усреднить
  - Вернуть `HourlyPoint[]`
- Заменить логику `storeAvgHourly` на этот бенчмарк когда `benchmarkWeekday` задан

**Фронтенд** (`SellerDetailView.tsx`):
- Передавать `benchmarkWeekday` = день недели из `dateFilter`
- Легенда графика: "Среднее (тот же день, 4 нед.)"

## 2. Сравнение по дням недели

**Фронтенд** (`SellerComparisonView.tsx`):
- Добавить переключатель режима: "Авто (текущий период)" | "По дням недели"
- В режиме "По дням недели":
  - Показать выбор дня недели (Пн–Вс)
  - Для выбранного дня: найти продавцов, работавших в этот день
  - Показать только тех, у кого есть смены в выбранный день
  - Возможность выбрать конкретные даты (если несколько четвергов в периоде)
- Если нет пересечений — показать ручной выбор периода (DateFilter)

**Бэкенд**:
- Добавить `weekday?: number` в параметры `computeSellerAdvancedStats`
- При фильтрации `daySessions` — оставить только дни с `dayOfWeek === weekday`

## 3. Фильтр рейтинга (today / all)

**Фронтенд** (`SellerList.tsx`):
- Добавить toggle: "Сегодня" | "Все за период"
- "Сегодня" (default): API с `since=until=today`, показывать только работающих сегодня
- "Все за период": API с выбранным диапазоном, показать всех
- Кнопка "развернуть" под списком, если режим "Сегодня" — переключает на "Все за период"

Фактически — это просто переключение DateFilter между "today" и "period".
